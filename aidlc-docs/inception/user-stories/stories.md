# 테이블오더 시스템 사용자 스토리

## 에픽 구조

### 에픽 1: 고객 주문 시스템 (Customer Ordering System)
고객이 테이블에서 직접 메뉴를 탐색하고 주문할 수 있는 시스템

### 에픽 2: 관리자 운영 시스템 (Admin Operations System)  
매장 직원과 관리자가 주문을 모니터링하고 관리할 수 있는 시스템

### 에픽 3: 시스템 관리 (System Management)
테이블 세션 관리, 메뉴 관리 등 시스템 운영을 위한 기능

# 사용자 스토리 (사용자 여정 기반 조직화)

## 🔵 에픽 1: 고객 주문 시스템

### 1.1 테이블 접근 및 자동 로그인

#### 스토리 1.1.1: 테이블 태블릿 초기 설정 (관리자용)
**사용자**: 매장 직원 (이영희)  
**우선순위**: MVP - 필수  
**페르소나**: 이영희 (매장 직원)

매장 직원으로서, 테이블 태블릿을 처음 설정할 때 매장 식별자, 테이블 번호, 테이블 비밀번호를 입력하여 해당 테이블에서 고객이 자동으로 로그인할 수 있도록 설정하고 싶다.

**수용 기준**:
- 매장 직원이 관리자 권한으로 테이블 태블릿에 접근할 수 있다
- 매장 식별자 입력 필드가 제공되고 유효성 검증이 수행된다
- 테이블 번호 입력 필드가 제공되고 중복 검증이 수행된다
- 테이블 비밀번호 입력 필드가 제공되고 보안 요구사항을 만족한다
- 설정 완료 후 로그인 정보가 로컬 스토리지에 안전하게 저장된다
- 16시간 세션이 자동으로 생성되고 활성화된다
- 설정 완료 후 고객용 메뉴 화면으로 자동 전환된다
- 설정 실패 시 명확한 오류 메시지가 표시된다

#### 스토리 1.1.2: 고객 자동 로그인
**사용자**: 고객 (김민수)  
**우선순위**: MVP - 필수  
**페르소나**: 김민수 (고객)

고객으로서, 테이블에 앉아 태블릿을 터치했을 때 별도의 로그인 절차 없이 자동으로 메뉴 화면에 접근하여 즉시 주문을 시작하고 싶다.

**수용 기준**:
- 태블릿 화면을 터치하면 즉시 반응한다
- 저장된 로그인 정보를 자동으로 불러와 인증을 수행한다
- 인증 성공 시 메뉴 화면이 3초 이내에 로드된다
- 로그인 과정에서 사용자에게 별도 입력을 요구하지 않는다
- 세션이 만료된 경우 자동으로 갱신을 시도한다
- 자동 로그인 실패 시 간단한 재시도 옵션을 제공한다
- 네트워크 오류 시 오프라인 모드 안내 메시지를 표시한다

### 1.2 메뉴 탐색 및 선택

#### 스토리 1.2.1: 메뉴 카테고리 탐색
**사용자**: 고객 (김민수)  
**우선순위**: MVP - 필수  
**페르소나**: 김민수 (고객)

고객으로서, 메뉴를 카테고리별로 분류하여 보고 원하는 카테고리를 쉽게 선택하여 해당 메뉴들을 탐색하고 싶다.

**수용 기준**:
- 메뉴 화면이 기본 화면으로 표시된다
- 카테고리 목록이 화면 상단 또는 좌측에 명확히 표시된다
- 각 카테고리는 터치하기 쉬운 크기(최소 44x44px)로 제공된다
- 카테고리 선택 시 해당 메뉴들이 1초 이내에 로드된다
- 현재 선택된 카테고리가 시각적으로 강조 표시된다
- 카테고리 간 전환이 부드럽고 빠르게 이루어진다
- 카테고리가 많을 경우 스크롤 또는 페이지네이션이 제공된다

#### 스토리 1.2.2: 메뉴 상세 정보 확인
**사용자**: 고객 (김민수)  
**우선순위**: MVP - 필수  
**페르소나**: 김민수 (고객)

고객으로서, 각 메뉴의 이름, 가격, 설명, 이미지를 명확하게 확인하여 주문할 메뉴를 결정하고 싶다.

**수용 기준**:
- 각 메뉴가 카드 형태로 표시된다
- 메뉴 이미지가 플레이스홀더로 제공된다 (MVP 범위)
- 메뉴명이 읽기 쉬운 크기와 폰트로 표시된다
- 가격이 명확하고 눈에 띄게 표시된다
- 메뉴 설명이 간결하고 이해하기 쉽게 작성된다
- 메뉴 카드 레이아웃이 일관성 있게 유지된다
- 터치 시 메뉴 선택 상태가 시각적으로 표시된다
- 품절 메뉴는 별도로 표시되고 선택이 비활성화된다

#### 스토리 1.2.3: 메뉴 검색 기능
**사용자**: 고객 (김민수)  
**우선순위**: 향후 기능  
**페르소나**: 김민수 (고객)

고객으로서, 특정 메뉴를 빠르게 찾기 위해 메뉴명으로 검색하고 싶다.

**수용 기준**:
- 검색 입력 필드가 화면 상단에 제공된다
- 입력과 동시에 실시간으로 검색 결과가 필터링된다
- 검색 결과가 없을 경우 안내 메시지가 표시된다
- 검색어 지우기 버튼이 제공된다
- 검색 중에도 카테고리 탐색이 가능하다

### 1.3 장바구니 관리

#### 스토리 1.3.1: 메뉴 장바구니 추가
**사용자**: 고객 (김민수)  
**우선순위**: MVP - 필수  
**페르소나**: 김민수 (고객)

고객으로서, 원하는 메뉴를 선택하여 장바구니에 추가하고 추가된 메뉴를 확인하고 싶다.

**수용 기준**:
- 메뉴 카드에 "장바구니 추가" 버튼이 명확히 표시된다
- 버튼 클릭 시 즉시 장바구니에 메뉴가 추가된다
- 장바구니 아이콘에 추가된 메뉴 개수가 표시된다
- 추가 완료 시 시각적 피드백(애니메이션, 색상 변화)이 제공된다
- 동일 메뉴 추가 시 수량이 자동으로 증가한다
- 장바구니 추가 실패 시 오류 메시지가 표시된다
- 장바구니 데이터가 로컬 스토리지에 저장된다

#### 스토리 1.3.2: 장바구니 수량 조절
**사용자**: 고객 (김민수)  
**우선순위**: MVP - 필수  
**페르소나**: 김민수 (고객)

고객으로서, 장바구니에 담긴 메뉴의 수량을 증가시키거나 감소시켜 원하는 수량으로 조절하고 싶다.

**수용 기준**:
- 각 메뉴 항목에 수량 증가(+) 버튼이 제공된다
- 각 메뉴 항목에 수량 감소(-) 버튼이 제공된다
- 현재 수량이 숫자로 명확히 표시된다
- 수량 변경 시 즉시 반영되고 총액이 자동 계산된다
- 수량이 1일 때 감소 버튼 클릭 시 메뉴가 장바구니에서 제거된다
- 수량 변경 시 시각적 피드백이 제공된다
- 최대 수량 제한이 있을 경우 안내 메시지가 표시된다

#### 스토리 1.3.3: 장바구니 총액 계산
**사용자**: 고객 (김민수)  
**우선순위**: MVP - 필수  
**페르소나**: 김민수 (고객)

고객으로서, 장바구니에 담긴 모든 메뉴의 총 금액을 실시간으로 확인하여 예산을 관리하고 싶다.

**수용 기준**:
- 장바구니 하단에 총액이 크고 명확하게 표시된다
- 메뉴 추가/삭제/수량 변경 시 총액이 즉시 업데이트된다
- 개별 메뉴의 소계(단가 × 수량)가 표시된다
- 총액 계산이 정확하고 오류가 없다
- 할인이나 부가세가 있을 경우 별도로 표시된다
- 총액이 0원일 때 적절한 안내 메시지가 표시된다

#### 스토리 1.3.4: 장바구니 메뉴 삭제
**사용자**: 고객 (김민수)  
**우선순위**: MVP - 필수  
**페르소나**: 김민수 (고객)

고객으로서, 장바구니에 잘못 담긴 메뉴나 더 이상 원하지 않는 메뉴를 개별적으로 삭제하고 싶다.

**수용 기준**:
- 각 메뉴 항목에 삭제 버튼(X 또는 휴지통 아이콘)이 제공된다
- 삭제 버튼 클릭 시 해당 메뉴가 즉시 장바구니에서 제거된다
- 삭제 후 총액이 자동으로 재계산된다
- 삭제 시 시각적 피드백(애니메이션)이 제공된다
- 실수로 삭제한 경우를 위한 "실행 취소" 옵션이 제공된다 (향후 기능)
- 마지막 메뉴 삭제 시 빈 장바구니 안내 메시지가 표시된다

#### 스토리 1.3.5: 장바구니 전체 비우기
**사용자**: 고객 (김민수)  
**우선순위**: MVP - 필수  
**페르소나**: 김민수 (고객)

고객으로서, 장바구니의 모든 메뉴를 한 번에 삭제하여 처음부터 다시 주문을 구성하고 싶다.

**수용 기준**:
- "장바구니 비우기" 버튼이 명확히 제공된다
- 버튼 클릭 시 확인 팝업이 표시된다
- 확인 후 모든 메뉴가 장바구니에서 제거된다
- 총액이 0원으로 초기화된다
- 비우기 완료 후 빈 장바구니 안내 메시지가 표시된다
- 실수 방지를 위한 확인 절차가 포함된다

#### 스토리 1.3.6: 장바구니 데이터 지속성
**사용자**: 고객 (김민수)  
**우선순위**: MVP - 필수  
**페르소나**: 김민수 (고객)

고객으로서, 브라우저를 새로고침하거나 실수로 다른 화면으로 이동했을 때도 장바구니 내용이 유지되어 다시 선택할 필요가 없기를 원한다.

**수용 기준**:
- 장바구니 데이터가 브라우저 로컬 스토리지에 저장된다
- 페이지 새로고침 후에도 장바구니 내용이 그대로 유지된다
- 브라우저 탭을 닫았다가 다시 열어도 장바구니가 복원된다
- 세션 만료 시에도 장바구니 데이터가 보존된다
- 주문 완료 후에만 장바구니가 자동으로 비워진다
- 데이터 저장 실패 시 사용자에게 알림이 제공된다

### 1.4 주문 생성 및 확정

#### 스토리 1.4.1: 주문 내역 최종 확인
**사용자**: 고객 (김민수)  
**우선순위**: MVP - 필수  
**페르소나**: 김민수 (고객)

고객으로서, 주문을 확정하기 전에 선택한 모든 메뉴와 수량, 총액을 최종적으로 확인하여 실수를 방지하고 싶다.

**수용 기준**:
- "주문하기" 버튼 클릭 시 주문 확인 화면이 표시된다
- 선택한 모든 메뉴가 목록으로 표시된다
- 각 메뉴의 이름, 수량, 단가, 소계가 명확히 표시된다
- 총 주문 금액이 크고 명확하게 표시된다
- 테이블 번호가 확인 화면에 표시된다
- "확정" 및 "취소" 버튼이 명확히 구분되어 제공된다
- 주문 내역이 정확하지 않을 경우 수정할 수 있는 옵션이 제공된다

#### 스토리 1.4.2: 주문 확정 처리
**사용자**: 고객 (김민수)  
**우선순위**: MVP - 필수  
**페르소나**: 김민수 (고객)

고객으로서, 주문 확인 후 "확정" 버튼을 클릭하여 주문을 서버에 전송하고 주문 번호를 받고 싶다.

**수용 기준**:
- "확정" 버튼 클릭 시 주문 데이터가 서버로 전송된다
- 주문 전송 중 로딩 인디케이터가 표시된다
- 주문 성공 시 고유한 주문 번호가 생성되어 표시된다
- 주문 번호가 크고 명확하게 표시된다
- 주문 완료 메시지가 표시된다
- 주문 정보에 매장 식별자, 테이블 번호, 세션 ID가 포함된다
- 주문 시각이 정확히 기록된다

#### 스토리 1.4.3: 주문 성공 후 처리
**사용자**: 고객 (김민수)  
**우선순위**: MVP - 필수  
**페르소나**: 김민수 (고객)

고객으로서, 주문이 성공적으로 완료된 후 장바구니가 자동으로 비워지고 메뉴 화면으로 돌아가서 추가 주문을 할 수 있기를 원한다.

**수용 기준**:
- 주문 완료 후 장바구니가 자동으로 비워진다
- 주문 번호 표시 후 5초 뒤 자동으로 메뉴 화면으로 리다이렉트된다
- 리다이렉트 전 카운트다운이 표시된다
- 사용자가 원할 경우 즉시 메뉴 화면으로 이동할 수 있는 버튼이 제공된다
- 메뉴 화면으로 돌아간 후 새로운 주문을 시작할 수 있다
- 주문 내역 확인 버튼이 제공되어 방금 주문한 내용을 확인할 수 있다

#### 스토리 1.4.4: 주문 실패 처리
**사용자**: 고객 (김민수)  
**우선순위**: MVP - 필수  
**페르소나**: 김민수 (고객)

고객으로서, 네트워크 오류나 서버 문제로 주문이 실패했을 때 명확한 안내를 받고 장바구니 내용을 잃지 않고 재시도할 수 있기를 원한다.

**수용 기준**:
- 주문 실패 시 명확한 오류 메시지가 표시된다
- 실패 원인이 사용자가 이해할 수 있는 언어로 설명된다
- 장바구니 내용이 그대로 유지된다
- "재시도" 버튼이 제공된다
- 네트워크 오류와 서버 오류가 구분되어 안내된다
- 지속적인 실패 시 매장 직원 호출 옵션이 제공된다
- 오류 로그가 기록되어 문제 해결에 활용된다

### 1.5 주문 내역 조회

#### 스토리 1.5.1: 현재 세션 주문 목록 조회
**사용자**: 고객 (김민수)  
**우선순위**: MVP - 필수  
**페르소나**: 김민수 (고객)

고객으로서, 현재 테이블 세션에서 주문한 모든 주문의 목록을 시간 순으로 확인하고 싶다.

**수용 기준**:
- "주문 내역" 메뉴 또는 버튼이 명확히 제공된다
- 현재 테이블 세션의 주문만 표시된다
- 주문이 시간 순(최신순)으로 정렬되어 표시된다
- 각 주문의 주문 번호, 주문 시각이 표시된다
- 주문 상태(대기중/준비중/완료)가 명확히 표시된다
- 이전 세션의 주문은 표시되지 않는다
- 주문이 없을 경우 안내 메시지가 표시된다

#### 스토리 1.5.2: 주문 상세 정보 확인
**사용자**: 고객 (김민수)  
**우선순위**: MVP - 필수  
**페르소나**: 김민수 (고객)

고객으로서, 특정 주문을 선택하여 주문한 메뉴 목록, 수량, 금액 등의 상세 정보를 확인하고 싶다.

**수용 기준**:
- 주문 목록에서 특정 주문을 클릭하면 상세 화면이 표시된다
- 해당 주문의 모든 메뉴가 목록으로 표시된다
- 각 메뉴의 이름, 수량, 단가, 소계가 표시된다
- 주문 총액이 명확히 표시된다
- 주문 시각과 주문 번호가 표시된다
- 현재 주문 상태가 표시된다
- 상세 화면에서 목록으로 돌아갈 수 있는 버튼이 제공된다

#### 스토리 1.5.3: 주문 상태 실시간 업데이트
**사용자**: 고객 (김민수)  
**우선순위**: 향후 기능  
**페르소나**: 김민수 (고객)

고객으로서, 주문 상태가 변경될 때마다 실시간으로 업데이트되어 음식이 언제쯤 준비될지 알고 싶다.

**수용 기준**:
- 관리자가 주문 상태를 변경하면 고객 화면에 즉시 반영된다
- 상태 변경 시 시각적 알림(색상 변화, 애니메이션)이 제공된다
- 주문 완료 시 별도의 알림이 표시된다
- 실시간 업데이트가 실패해도 새로고침으로 최신 상태를 확인할 수 있다
- 네트워크 연결 상태에 따른 적절한 안내가 제공된다

## 🟢 에픽 2: 관리자 운영 시스템

### 2.1 관리자 인증 및 접근

#### 스토리 2.1.1: 매장 관리자 로그인
**사용자**: 매장 직원 (이영희), 매장 관리자 (박사장)  
**우선순위**: MVP - 필수  
**페르소나**: 이영희, 박사장

매장 관리자로서, 매장 식별자와 사용자 인증 정보를 입력하여 관리자 시스템에 안전하게 로그인하고 싶다.

**수용 기준**:
- 관리자 로그인 화면이 고객 화면과 명확히 구분된다
- 매장 식별자 입력 필드가 제공된다
- 사용자명 입력 필드가 제공된다
- 비밀번호 입력 필드가 제공되고 마스킹 처리된다
- 로그인 버튼 클릭 시 서버 인증이 수행된다
- 인증 성공 시 관리자 대시보드로 이동한다
- 인증 실패 시 명확한 오류 메시지가 표시된다
- 로그인 시도 제한 기능이 적용된다

#### 스토리 2.1.2: 관리자 세션 관리
**사용자**: 매장 직원 (이영희), 매장 관리자 (박사장)  
**우선순위**: MVP - 필수  
**페르소나**: 이영희, 박사장

매장 관리자로서, 로그인 후 16시간 동안 세션이 유지되어 반복적인 로그인 없이 업무를 수행하고 싶다.

**수용 기준**:
- 로그인 성공 시 16시간 유효한 JWT 토큰이 생성된다
- 브라우저 새로고침 시에도 세션이 유지된다
- 세션 만료 15분 전 갱신 알림이 표시된다
- 세션 만료 시 자동으로 로그인 화면으로 이동한다
- 수동 로그아웃 기능이 제공된다
- 세션 정보가 안전하게 저장되고 관리된다
- 동시 로그인 세션 수가 제한된다

### 2.2 실시간 주문 모니터링

#### 스토리 2.2.1: 주문 대시보드 실시간 업데이트
**사용자**: 매장 직원 (이영희), 주방 담당자 (정주방장)  
**우선순위**: MVP - 필수  
**페르소나**: 이영희, 정주방장

매장 직원으로서, 새로운 주문이 들어올 때마다 실시간으로 대시보드에 표시되어 즉시 확인하고 처리할 수 있기를 원한다.

**수용 기준**:
- Server-Sent Events(SSE)를 통해 실시간 주문 업데이트가 수신된다
- 새 주문이 2초 이내에 대시보드에 표시된다
- 테이블별로 카드 형태로 주문이 그룹화되어 표시된다
- 새 주문 시 시각적 강조(색상 변화, 애니메이션)가 적용된다
- 주문 카드에 테이블 번호, 주문 시각, 총액이 표시된다
- 연결 끊김 시 자동 재연결이 시도된다
- 실시간 업데이트 실패 시 수동 새로고침 옵션이 제공된다

#### 스토리 2.2.2: 테이블별 주문 현황 표시
**사용자**: 매장 직원 (이영희), 매장 관리자 (박사장)  
**우선순위**: MVP - 필수  
**페르소나**: 이영희, 박사장

매장 관리자로서, 각 테이블의 현재 주문 상황을 그리드 형태로 한눈에 파악하여 효율적으로 매장을 운영하고 싶다.

**수용 기준**:
- 테이블별로 카드가 그리드 형태로 배치된다
- 각 테이블 카드에 테이블 번호가 명확히 표시된다
- 테이블별 총 주문액이 표시된다
- 최신 주문 3개가 미리보기로 표시된다
- 주문이 없는 테이블은 "주문 없음" 상태로 표시된다
- 테이블 상태(활성/비활성)가 시각적으로 구분된다
- 테이블 카드 크기가 일관성 있게 유지된다

#### 스토리 2.2.3: 주문 상세 정보 조회
**사용자**: 매장 직원 (이영희), 주방 담당자 (정주방장)  
**우선순위**: MVP - 필수  
**페르소나**: 이영희, 정주방장

매장 직원으로서, 테이블 카드를 클릭하여 해당 테이블의 모든 주문 내역과 메뉴 상세 정보를 확인하고 싶다.

**수용 기준**:
- 테이블 카드 클릭 시 상세 화면이 모달 또는 새 화면으로 표시된다
- 해당 테이블의 모든 주문이 시간순으로 표시된다
- 각 주문의 전체 메뉴 목록이 표시된다
- 메뉴별 수량과 특별 요청사항이 표시된다
- 주문 상태(대기중/준비중/완료)가 명확히 표시된다
- 상세 화면에서 대시보드로 돌아갈 수 있는 버튼이 제공된다
- 상세 정보 로딩이 빠르게 이루어진다

#### 스토리 2.2.4: 주문 상태 변경
**사용자**: 매장 직원 (이영희), 주방 담당자 (정주방장)  
**우선순위**: MVP - 필수  
**페르소나**: 이영희, 정주방장

매장 직원으로서, 주문의 진행 상황에 따라 상태를 "대기중", "준비중", "완료"로 변경하여 고객과 팀원들이 현재 상황을 파악할 수 있도록 하고 싶다.

**수용 기준**:
- 각 주문에 상태 변경 드롭다운 또는 버튼이 제공된다
- 상태 옵션: 대기중, 준비중, 완료가 제공된다
- 상태 변경 시 즉시 서버에 반영된다
- 상태 변경 시 시각적 피드백이 제공된다
- 변경된 상태가 고객 화면에도 실시간으로 반영된다
- 상태 변경 실패 시 오류 메시지가 표시된다
- 상태 변경 이력이 기록된다

#### 스토리 2.2.5: 주문 알림 및 강조 표시
**사용자**: 매장 직원 (이영희), 주방 담당자 (정주방장)  
**우선순위**: MVP - 필수  
**페르소나**: 이영희, 정주방장

매장 직원으로서, 새로운 주문이나 긴급한 주문을 놓치지 않도록 시각적 알림과 강조 표시를 받고 싶다.

**수용 기준**:
- 새 주문 시 해당 테이블 카드가 특별한 색상으로 강조된다
- 주문 후 일정 시간이 지나면 대기 시간 경고가 표시된다
- 알림 색상이 주문 상태별로 구분된다 (신규: 파란색, 지연: 빨간색)
- 알림 강조가 점진적으로 사라진다 (페이드 아웃)
- 브라우저 알림 권한이 있을 경우 데스크톱 알림이 표시된다
- 알림 설정을 사용자가 조정할 수 있다 (향후 기능)

### 2.3 테이블 관리

#### 스토리 2.3.1: 테이블 초기 설정 관리
**사용자**: 매장 직원 (이영희), 매장 관리자 (박사장)  
**우선순위**: MVP - 필수  
**페르소나**: 이영희, 박사장

매장 관리자로서, 새로운 테이블을 시스템에 등록하고 테이블 번호와 비밀번호를 설정하여 고객이 사용할 수 있도록 준비하고 싶다.

**수용 기준**:
- 테이블 관리 메뉴가 관리자 인터페이스에 제공된다
- 새 테이블 추가 기능이 제공된다
- 테이블 번호 설정 및 중복 검증이 수행된다
- 테이블 비밀번호 생성 및 설정 기능이 제공된다
- 설정 완료 후 16시간 세션이 자동 생성된다
- 테이블 설정 정보가 안전하게 저장된다
- 설정 완료 시 확인 메시지가 표시된다

#### 스토리 2.3.2: 주문 직권 삭제
**사용자**: 매장 직원 (이영희), 매장 관리자 (박사장)  
**우선순위**: MVP - 필수  
**페르소나**: 이영희, 박사장

매장 직원으로서, 고객 요청이나 실수로 인해 특정 주문을 삭제하고 테이블 총 주문액을 재계산하고 싶다.

**수용 기준**:
- 각 주문에 삭제 버튼이 제공된다
- 삭제 버튼 클릭 시 확인 팝업이 표시된다
- 확인 후 주문이 즉시 삭제된다
- 삭제된 주문이 테이블 총액에서 차감된다
- 삭제 완료 시 성공 메시지가 표시된다
- 삭제 실패 시 오류 메시지가 표시된다
- 삭제 이력이 기록되어 추적 가능하다

#### 스토리 2.3.3: 테이블 세션 종료 (매장 이용 완료)
**사용자**: 매장 직원 (이영희), 매장 관리자 (박사장)  
**우선순위**: MVP - 필수  
**페르소나**: 이영희, 박사장

매장 직원으로서, 고객이 식사를 마치고 떠날 때 해당 테이블의 세션을 종료하여 다음 고객을 위해 테이블을 초기화하고 싶다.

**수용 기준**:
- 각 테이블에 "이용 완료" 버튼이 제공된다
- 버튼 클릭 시 확인 팝업이 표시된다
- 확인 후 해당 세션의 모든 주문이 과거 이력으로 이동한다
- 테이블의 현재 주문 목록이 0으로 리셋된다
- 테이블 총 주문액이 0원으로 초기화된다
- 새로운 세션이 준비 상태로 설정된다
- 완료 처리 시 성공 메시지가 표시된다

#### 스토리 2.3.4: 과거 주문 내역 조회
**사용자**: 매장 관리자 (박사장), 매장 직원 (이영희)  
**우선순위**: MVP - 필수  
**페르소나**: 박사장, 이영희

매장 관리자로서, 특정 테이블의 과거 주문 내역을 조회하여 매출 분석이나 고객 문의에 대응하고 싶다.

**수용 기준**:
- 각 테이블에 "과거 내역" 버튼이 제공된다
- 버튼 클릭 시 과거 주문 목록이 모달로 표시된다
- 과거 주문이 시간 역순으로 정렬되어 표시된다
- 각 주문의 번호, 시각, 메뉴 목록, 총액이 표시된다
- 매장 이용 완료 시각이 표시된다
- 날짜 필터링 기능이 제공된다

## 🔧 에픽 3: 시스템 관리

### 3.1 메뉴 관리

#### 스토리 3.1.1: 메뉴 목록 조회
**사용자**: 매장 관리자 (박사장), 매장 직원 (이영희)  
**우선순위**: MVP - 필수  
**페르소나**: 박사장, 이영희

매장 관리자로서, 현재 등록된 모든 메뉴를 카테고리별로 조회하여 메뉴 현황을 파악하고 싶다.

**수용 기준**:
- 메뉴 관리 화면이 관리자 인터페이스에 제공된다
- 메뉴가 카테고리별로 그룹화되어 표시된다
- 각 메뉴의 이름, 가격, 설명이 표시된다
- 메뉴 상태(활성/비활성)가 표시된다
- 메뉴 목록이 빠르게 로드된다
- 검색 기능이 제공된다 (향후 기능)
- 카테고리별 필터링이 가능하다

#### 스토리 3.1.2: 새 메뉴 등록
**사용자**: 매장 관리자 (박사장), 매장 직원 (이영희)  
**우선순위**: MVP - 필수  
**페르소나**: 박사장, 이영희

매장 관리자로서, 새로운 메뉴를 시스템에 등록하여 고객이 주문할 수 있도록 하고 싶다.

**수용 기준**:
- "새 메뉴 추가" 버튼이 제공된다
- 메뉴명 입력 필드가 제공되고 필수 검증이 수행된다
- 가격 입력 필드가 제공되고 숫자 검증이 수행된다
- 메뉴 설명 입력 필드가 제공된다
- 카테고리 선택 드롭다운이 제공된다
- 이미지 URL 입력 필드가 제공된다 (MVP에서는 플레이스홀더 사용)
- 저장 버튼 클릭 시 데이터 검증이 수행된다
- 등록 성공 시 메뉴 목록에 즉시 반영된다

#### 스토리 3.1.3: 메뉴 정보 수정
**사용자**: 매장 관리자 (박사장), 매장 직원 (이영희)  
**우선순위**: MVP - 필수  
**페르소나**: 박사장, 이영희

매장 관리자로서, 기존 메뉴의 정보를 수정하여 가격 변경이나 설명 업데이트를 반영하고 싶다.

**수용 기준**:
- 각 메뉴에 "수정" 버튼이 제공된다
- 수정 화면에 기존 정보가 미리 채워진다
- 모든 필드를 수정할 수 있다
- 수정 시 데이터 검증이 수행된다
- 저장 시 변경사항이 즉시 반영된다
- 수정 취소 기능이 제공된다
- 수정 완료 시 확인 메시지가 표시된다

#### 스토리 3.1.4: 메뉴 삭제
**사용자**: 매장 관리자 (박사장)  
**우선순위**: MVP - 필수  
**페르소나**: 박사장

매장 관리자로서, 더 이상 제공하지 않는 메뉴를 시스템에서 삭제하여 고객이 주문할 수 없도록 하고 싶다.

**수용 기준**:
- 각 메뉴에 "삭제" 버튼이 제공된다
- 삭제 버튼 클릭 시 확인 팝업이 표시된다
- 확인 후 메뉴가 시스템에서 제거된다
- 삭제된 메뉴는 고객 화면에서 즉시 사라진다
- 진행 중인 주문에 포함된 메뉴는 삭제 시 경고가 표시된다
- 삭제 완료 시 확인 메시지가 표시된다
- 삭제 이력이 기록된다

#### 스토리 3.1.5: 메뉴 노출 순서 조정
**사용자**: 매장 관리자 (박사장)  
**우선순위**: 향후 기능  
**페르소나**: 박사장

매장 관리자로서, 메뉴의 표시 순서를 조정하여 인기 메뉴나 추천 메뉴를 상단에 배치하고 싶다.

**수용 기준**:
- 드래그 앤 드롭으로 메뉴 순서를 변경할 수 있다
- 순서 변경이 즉시 고객 화면에 반영된다
- 카테고리 내에서만 순서 변경이 가능하다
- 순서 변경 완료 시 자동 저장된다
- 순서 초기화 기능이 제공된다

- 고객의 장바구니 데이터가 로컬에 보존된다

- 오프라인 모드에서 가능한 기능과 불가능한 기능이 명확히 구분된다
- 연결 복구 알림이 표시된다

#### 스토리 E.2: 서버 오류 처리
**사용자**: 고객 (김민수), 매장 직원 (이영희)

**페르소나**: 김민수, 이영희

**수용 기준**:
- 서버 오류 시 사용자 친화적인 오류 메시지가 표시된다
- 오류 유형별로 다른 안내 메시지가 제공된다
- 재시도 버튼이 제공된다
- 지속적인 오류 시 매장 직원 호출 옵션이 제공된다
- 오류 로그가 자동으로 기록된다
- 오류 복구 시 자동으로 정상 화면으로 복귀한다

### 세션 및 인증 오류

#### 스토리 E.3: 세션 만료 처리
**사용자**: 고객 (김민수), 매장 직원 (이영희)

**수용 기준**:
- 세션 만료 15분 전 갱신 알림이 표시된다
- 고객의 경우 자동으로 세션이 갱신된다
- 관리자의 경우 재로그인 화면이 표시된다
- 세션 만료 시 진행 중인 작업이 임시 저장된다
- 재인증 후 이전 작업을 계속할 수 있다
- 세션 갱신 실패 시 적절한 안내가 제공된다

### 데이터 및 입력 오류

#### 스토리 E.4: 잘못된 데이터 입력 처리
**사용자**: 고객 (김민수), 매장 직원 (이영희)  
**우선순위**: MVP - 필수  
**페르소나**: 김민수, 이영희

사용자로서, 잘못된 데이터를 입력했을 때 명확한 안내를 받고 쉽게 수정할 수 있기를 원한다.

**수용 기준**:
- 입력 필드별로 실시간 검증이 수행된다
- 오류가 있는 필드가 시각적으로 강조된다
- 구체적인 오류 메시지가 필드 근처에 표시된다
- 올바른 입력 형식의 예시가 제공된다
- 오류 수정 후 즉시 검증 상태가 업데이트된다
- 전체 폼 제출 전 모든 오류가 해결되어야 한다

#### 스토리 E.5: 동시성 충돌 처리
**사용자**: 매장 직원 (이영희), 주방 담당자 (정주방장)  
**우선순위**: MVP - 필수  
**페르소나**: 이영희, 정주방장

매장 직원으로서, 여러 직원이 동시에 같은 주문을 처리하려 할 때 충돌을 방지하고 일관성을 유지하고 싶다.

**수용 기준**:
- 주문 상태 변경 시 최신 상태를 먼저 확인한다
- 동시 수정 시도 시 충돌 감지 메시지가 표시된다
- 충돌 발생 시 최신 데이터로 화면이 갱신된다
- 사용자가 변경사항을 다시 확인하고 적용할 수 있다
- 중요한 작업(주문 삭제, 세션 종료)은 추가 확인이 필요하다
- 충돌 해결 후 정상적으로 작업을 계속할 수 있다

## INVEST 기준 검증

모든 사용자 스토리는 다음 INVEST 기준을 만족합니다:

- **Independent (독립적)**: 각 스토리는 다른 스토리와 독립적으로 개발 가능
- **Negotiable (협상 가능)**: 구현 방법은 개발팀과 협의하여 조정 가능
- **Valuable (가치 있는)**: 각 스토리는 사용자에게 명확한 가치 제공
- **Estimable (추정 가능)**: 개발 시간과 복잡도 추정 가능
- **Small (작은 단위)**: 1-2주 내 완료 가능한 크기로 분할
- **Testable (테스트 가능)**: 명확한 수용 기준으로 테스트 가능
