# 테이블오더 시스템 페르소나

## 페르소나 1: 김민수 (고객)

### 기본 정보
- **이름**: 김민수
- **나이**: 35세
- **직업**: 회사원
- **기술 숙련도**: 일반적인 사용자 (기본적인 터치 조작 가능)

### 특성
- 스마트폰 사용에 익숙하지만 복잡한 앱은 어려워함
- 직관적이고 단순한 인터페이스를 선호
- 빠르고 편리한 서비스를 중요하게 생각
- 대기 시간을 싫어하고 효율성을 추구

### 목표
- 별도 로그인 없이 빠르게 주문하고 싶음
- 메뉴를 쉽게 찾고 비교하고 싶음
- 주문 실수 없이 정확하게 주문하고 싶음
- 주문 상태를 확인하고 싶음

### 동기
- 웨이터를 기다리지 않고 즉시 주문
- 메뉴판을 넘기며 찾는 번거로움 해소
- 주문 내역을 명확히 확인하여 실수 방지
- 음식이 언제 나오는지 알고 싶음

### 제약사항 및 우려사항
- 복잡한 인터페이스는 사용하기 어려움
- 터치 버튼이 너무 작으면 잘못 누를 수 있음
- 시스템 오류 시 어떻게 해야 할지 모름
- 개인정보 입력을 꺼려함

### 사용 환경
- 카페나 레스토랑의 테이블에 비치된 태블릿
- 가족이나 친구와 함께 식사하는 상황
- 조명이 다양한 매장 환경
- 소음이 있는 환경에서 시각적 피드백 중요

### 기대하는 경험
- 태블릿을 켜면 바로 메뉴가 보임
- 카테고리별로 메뉴가 잘 정리되어 있음
- 이미지와 설명으로 메뉴를 쉽게 이해
- 장바구니에서 주문 내역을 쉽게 수정
- 주문 완료 후 명확한 확인 메시지

- **나이**: 28세
- **직업**: 카페 직원
- **기술 숙련도**: 기본적인 컴퓨터 사용 가능

- 매장 운영 업무에 집중
- 고객 서비스를 중요하게 생각
- 빠른 주문 처리를 통한 효율성 추구
- 단순하고 직관적인 시스템 선호

### 목표
- 들어오는 주문을 빠르게 확인하고 처리
- 주문 상태를 실시간으로 업데이트
- 고객 문의에 신속하게 대응
- 매장 운영을 원활하게 지원

### 동기
- 고객 대기 시간 단축
- 주문 실수 방지
- 업무 효율성 향상
- 고객 만족도 증대

### 제약사항 및 우려사항
- 복잡한 시스템은 학습하기 어려움

- 주문 누락이나 실수가 발생할까 걱정
- 기술적 문제 발생 시 해결 방법을 모름

### 사용 환경
- 매장 내 카운터나 주방 근처의 컴퓨터/태블릿
- 바쁜 시간대에 여러 업무를 동시에 처리
- 고객과 대화하면서 시스템 조작
- 소음과 움직임이 많은 작업 환경

### 기대하는 경험
- 새 주문이 들어오면 즉시 알림
- 테이블별로 주문 현황을 한눈에 파악
- 주문 상태를 쉽게 변경 가능
- 문제 발생 시 간단한 해결 방법 제공

# 사용자 여정 매핑

## 고객 주문 여정 (김민수 페르소나)

### 여정 단계
1. **테이블 도착** → 2. **시스템 접근** → 3. **메뉴 탐색** → 4. **장바구니 관리** → 5. **주문 확정** → 6. **주문 확인**

### 상세 여정

#### 1단계: 테이블 도착
- **행동**: 테이블에 앉아 태블릿 확인
- **생각**: "이걸 어떻게 사용하지?"
- **감정**: 호기심, 약간의 불안
- **터치포인트**: 테이블 태블릿
- **기대**: 사용법이 직관적이길 바람

#### 2단계: 시스템 접근 (자동 로그인)
- **행동**: 태블릿 화면 터치
- **생각**: "로그인 없이 바로 되네"
- **감정**: 안도감, 편리함
- **터치포인트**: 자동 로그인 시스템
- **기대**: 빠른 접근, 복잡한 절차 없음

#### 3단계: 메뉴 탐색
- **행동**: 카테고리별 메뉴 확인, 이미지와 설명 읽기
- **생각**: "이 메뉴가 맛있을까? 가격은 적당한가?"
- **감정**: 선택의 즐거움, 때로는 고민
- **터치포인트**: 메뉴 카테고리, 메뉴 카드
- **기대**: 명확한 정보, 매력적인 이미지

#### 4단계: 장바구니 관리
- **행동**: 메뉴 선택, 수량 조절, 총액 확인
- **생각**: "총 얼마나 나올까? 이 정도면 적당하네"
- **감정**: 통제감, 만족감
- **터치포인트**: 장바구니 인터페이스
- **기대**: 쉬운 수정, 명확한 총액 표시

#### 5단계: 주문 확정
- **행동**: 최종 확인 후 주문 버튼 클릭
- **생각**: "잘못 주문한 건 없나? 확실히 주문됐나?"
- **감정**: 약간의 긴장, 기대
- **터치포인트**: 주문 확정 버튼
- **기대**: 명확한 확인 과정

#### 6단계: 주문 확인
- **행동**: 주문 번호 확인, 주문 내역 재확인
- **생각**: "주문이 잘 들어갔구나. 언제쯤 나올까?"
- **감정**: 안심, 기대
- **터치포인트**: 주문 확인 화면
- **기대**: 명확한 주문 번호, 예상 시간

### 예외 상황 및 오류 시나리오
- **네트워크 오류**: 주문 중 연결 끊김 → 장바구니 유지, 재시도 안내
- **잘못된 선택**: 메뉴 잘못 선택 → 쉬운 수정 기능
- **시스템 오류**: 주문 처리 실패 → 명확한 오류 메시지, 해결 방법 안내
- **세션 만료**: 오랜 시간 방치 → 자동 갱신 또는 재로그인 안내

## 관리자 운영 여정 (이영희, 박사장, 정주방장 페르소나)

### 여정 단계
1. **시스템 로그인** → 2. **대시보드 확인** → 3. **주문 모니터링** → 4. **주문 처리** → 5. **상태 업데이트** → 6. **완료 처리**

### 상세 여정

#### 1단계: 시스템 로그인
- **행동**: 매장 식별자, 사용자명, 비밀번호 입력
- **생각**: "오늘도 바쁜 하루가 시작되겠네"
- **감정**: 일상적, 준비된 마음
- **터치포인트**: 로그인 화면
- **기대**: 빠른 로그인, 세션 유지

#### 2단계: 대시보드 확인
- **행동**: 전체 테이블 현황, 대기 주문 확인
- **생각**: "어느 테이블이 바쁘고, 얼마나 많은 주문이 들어왔나?"
- **감정**: 상황 파악, 업무 계획
- **터치포인트**: 관리자 대시보드
- **기대**: 한눈에 보이는 현황, 실시간 업데이트

#### 3단계: 주문 모니터링 (실시간)
- **행동**: 새 주문 알림 확인, 주문 내용 검토
- **생각**: "새 주문이 들어왔네. 어떤 메뉴인가?"
- **감정**: 집중, 책임감
- **터치포인트**: 실시간 알림, 주문 카드
- **기대**: 즉시 알림, 명확한 주문 정보

#### 4단계: 주문 처리
- **행동**: 주문 상세 확인, 주방에 전달, 상태 변경
- **생각**: "이 주문을 빠르게 처리해야겠다"
- **감정**: 효율성 추구, 고객 서비스 마인드
- **터치포인트**: 주문 상세 화면, 상태 변경 버튼
- **기대**: 쉬운 상태 변경, 명확한 처리 과정

#### 5단계: 상태 업데이트
- **행동**: 조리 중, 완료 등 상태 실시간 업데이트
- **생각**: "고객이 언제쯤 받을 수 있는지 알려줘야지"
- **감정**: 고객 배려, 서비스 품질 의식
- **터치포인트**: 상태 업데이트 인터페이스
- **기대**: 간단한 조작, 즉시 반영

#### 6단계: 완료 처리
- **행동**: 서빙 완료 후 주문 완료 처리, 테이블 정리
- **생각**: "이 테이블은 끝났고, 다음 주문을 준비하자"
- **감정**: 성취감, 다음 업무 준비
- **터치포인트**: 완료 처리 버튼
- **기대**: 명확한 완료 표시, 자동 정리

### 관리자별 특화 여정

#### 매장 직원 (이영희) 특화
- **주문 삭제**: 고객 요청 시 주문 삭제 → 확인 팝업 → 총액 재계산
- **테이블 세션 관리**: 고객 퇴장 시 세션 종료 → 과거 이력 이동

#### 매장 관리자 (박사장) 특화
- **과거 내역 조회**: 매출 분석을 위한 과거 주문 데이터 확인
- **전체 현황 모니터링**: 매장 전체 운영 상황 파악

#### 주방 담당자 (정주방장) 특화
- **조리 우선순위**: 주문 시간과 메뉴 특성에 따른 조리 순서 결정
- **조리 완료 알림**: 음식 완료 시 서빙 직원에게 알림

### 예외 상황 및 오류 시나리오
- **시스템 오류**: 주문 표시 오류 → 수동 확인 절차, 기술 지원 요청
- **주문 변경/취소**: 고객 요청 시 → 확인 절차, 주방 통보
- **네트워크 장애**: 실시간 업데이트 중단 → 수동 새로고침, 백업 절차
- **과부하 상황**: 주문 폭주 시 → 우선순위 관리, 대기 시간 안내
- **직원 교체**: 근무 교대 시 → 인수인계 절차, 세션 이관

## 터치포인트 및 상호작용 지점

### 고객 터치포인트
1. **테이블 태블릿** - 주요 상호작용 지점
2. **자동 로그인 시스템** - 첫 접근 경험
3. **메뉴 인터페이스** - 선택과 탐색
4. **장바구니** - 주문 관리
5. **주문 확인 화면** - 최종 확인

### 관리자 터치포인트
1. **로그인 시스템** - 접근 제어
2. **관리자 대시보드** - 전체 현황 파악
3. **실시간 알림** - 새 주문 인지
4. **주문 관리 인터페이스** - 상태 변경 및 처리
5. **과거 데이터 조회** - 분석 및 관리

### 시스템 간 연결점
- **실시간 동기화**: 고객 주문 → 관리자 알림
- **상태 업데이트**: 관리자 처리 → 고객 확인
- **데이터 저장**: 모든 활동 → 과거 이력 보관
